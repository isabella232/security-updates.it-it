Bollettino Microsoft sulla sicurezza MS14-025 - Importante
==========================================================

Una vulnerabilità nelle preferenze Criteri di gruppo può consentire l'acquisizione di privilegi più elevati (2962486)
---------------------------------------------------------------------------------------------------------------------

Data di pubblicazione: 13 maggio 2014

**Versione:** 1.0

Informazioni generali
---------------------

### Riepilogo

Questo aggiornamento per la protezione risolve una vulnerabilità divulgata pubblicamente in Microsoft Windows. La vulnerabilità può consentire l'acquisizione di privilegi più elevati se si utilizzano le preferenze Criteri di gruppo Active Directory per distribuire le password all'interno del dominio; questa procedura può consentire a un utente malintenzionato di recuperare e decrittografare la password memorizzata con le preferenze Criteri di gruppo.

Questo aggiornamento per la protezione è considerato di livello importante per gli Strumenti di amministrazione remota del server se installati sulle edizioni interessate di Windows Vista, Windows 7, Windows 8 e Windows 8.1. L'aggiornamento è anche considerato di livello importante per tutte le edizioni interessate di Windows Server 2008, Windows Server 2008 R2, Windows Server 2012 e Windows Server 2012 R2. Per ulteriori informazioni, vedere la sezione **Software interessato e Software non interessato**.

L'aggiornamento per la protezione risolve la vulnerabilità eliminando la possibilità di configurare e distribuire password che utilizzano determinate estensioni di preferenze Criteri di gruppo. Per ulteriori informazioni sulla vulnerabilità, cercare la voce relativa alla vulnerabilità specifica nella sottosezione Domande frequenti più avanti in questo bollettino.

**Altre operazioni richieste.** Per essere completamente protetti dalla vulnerabilità descritta in questo bollettino, i clienti con criteri di gruppo esistenti che utilizzano le preferenze Criteri di gruppo dovranno eseguire ulteriori operazioni per rimuovere tali criteri. Per ulteriori informazioni, vedere le **Domande frequenti sulla vulnerabilità** in questo bollettino.

**Raccomandazione.** Microsoft consiglia ai clienti di applicare l'aggiornamento prima possibile e di effettuare le azioni richieste descritte in precedenza e nelle Domande frequenti correlate.

L'aggiornamento è disponibile per il download solo nell'[Area download Microsoft](http://www.microsoft.com/downloads/results.aspx?displaylang=it&freetext=security%20update) e nel [catalogo di Microsoft Update](http://catalog.update.microsoft.com/v7/site/home.aspx).

Vedere anche la sezione, **Strumenti e informazioni sul rilevamento e sulla distribuzione**, riportata di seguito nel presente bollettino.

Domande frequenti sull'aggiornamento  
------------------------------------
  
<span id="sectionToggle1"></span>
**Quali modifiche vengono apportate da questo aggiornamento alle preferenze Criteri di gruppo?   
**Per informazioni dettagliate sulle modifiche apportate dall'aggiornamento al funzionamento delle preferenze Criteri di gruppo, consultare l'[articolo della Microsoft Knowledge Base 2962486](https://support.microsoft.com/kb/2962486).
  
**Perché gli aggiornamenti sono disponibili solo dall'Area download Microsoft e dal catalogo di Microsoft Update?**   
Microsoft rilascia questi aggiornamenti nell'[Area download Microsoft](http://www.microsoft.com/downloads/results.aspx?displaylang=it&freetext=security%20update) e nel [catalogo di Microsoft Update](http://catalog.update.microsoft.com/v7/site/home.aspx) in modo che i clienti possano iniziare ad aggiornare i propri sistemi prima possibile.
  
**L'aggiornamento include modifiche di funzionalità relative alla sicurezza?**   
Sì. L'aggiornamento modifica la finestra Editor Gestione Criteri di gruppo della Console Gestione Criteri di gruppo (GPMC) eliminando la possibilità di configurare e distribuire le password tramite le seguenti estensioni delle preferenze Criteri di gruppo:
  
-   Gruppo e utente locale  
-   Unità mappate  
-   Servizi  
-   Attività pianificate (livello superiore)  
-   Attività pianificate (livello inferiore)  
-   Attività immediate (livello superiore)  
-   Attività immediate (livello inferiore)  
-   Origini dati
  
**Sono disponibili più aggiornamenti per Windows 8.1 e Windows Server 2012 R2. È necessario installare tutti gli aggiornamenti?**   
No. A seconda della configurazione del sistema per la ricezione degli aggiornamenti, solo uno degli aggiornamenti disponibili per una data edizione di Windows 8.1 o Windows Server 2012 R2 è applicabile.
  
Per i sistemi che eseguono Windows 8.1 o Windows Server 2012 R2:
  
-   L'aggiornamento 2928120 è applicabile ai sistemi su cui è installato l'aggiornamento 2919355.  
-   L'aggiornamento 2961899 è applicabile ai sistemi su cui non è installato l'aggiornamento 2919355.
  
**Se si esegue Windows 8.1 o Windows Server 2012 R2, esiste un prerequisito per l'installazione dell'aggiornamento 2928120?**   
Sì. I clienti che eseguono i sistemi Windows 8.1 o Windows Server 2012 R2 devono installare l'aggiornamento 2919355 rilasciato ad aprile 2014 prima dell'aggiornamento 2928120. Per ulteriori informazioni su questo aggiornamento necessario come prerequisito, consultare l'[articolo della Microsoft Knowledge Base 2919355](http://support.microsoft.com/kb/2919355).
  
**In che modo le installazioni Server Core sono interessate dalle vulnerabilità risolte in questo bollettino?**   
Le vulnerabilità risolte da questo aggiornamento non interessano le edizioni supportate di Windows Server 2008, Windows Server 2008 R2, Windows Server 2012 o Windows Server 2012 R2 come indicato nella tabella Software non interessato, installate utilizzando l'opzione di installazione Server Core. Per ulteriori informazioni su questa modalità di installazione, vedere gli articoli di TechNet [Gestione di un'installazione Server Core: Panoramica](http://technet.microsoft.com/library/ee441255), [Manutenzione di un'installazione Server Core](http://technet.microsoft.com/library/ff698994) e [Panoramica di Server Core e dell'integrazione server completa](http://technet.microsoft.com/library/hh831758).
  
**Se si utilizza una versione precedente del software discusso nel presente bollettino, come bisogna comportarsi?**   
I prodotti software elencati in questo bollettino sono stati sottoposti a test per determinare quali versioni sono interessate dalla vulnerabilità. Le altre versioni sono al termine del ciclo di vita del supporto. Per ulteriori informazioni sul ciclo di vita dei prodotti, visitare il sito Web [Ciclo di vita del supporto Microsoft](http://support.microsoft.com/common/international.aspx?rdpath=gp;%5Bln%5D;lifecycle).
  
Per evitare di esporre i propri sistemi a potenziali vulnerabilità, si raccomanda ai clienti che utilizzano versioni precedenti del software di eseguire quanto prima la migrazione a versioni supportate. Per informazioni sulla disponibilità del supporto per la versione del software in uso, vedere [Selezionare un prodotto per informazioni sul ciclo di vita](http://support.microsoft.com/gp/lifeselect). Per ulteriori informazioni sui service pack relativi a queste versioni del software, vedere [Criteri relativi al ciclo di vita del supporto per i service pack](http://support.microsoft.com/gp/lifesupsps).
  
I clienti che hanno ancora la necessità di servizi di supporto per versioni precedenti del software sono invitati a contattare il loro rappresentante del team Microsoft, il responsabile tecnico dei rapporti con i clienti oppure il rappresentante del partner Microsoft appropriato per informazioni sui servizi di supporto personalizzati. I clienti che non dispongono di un contratto Premier o Authorized possono contattare le filiali Microsoft locali. Per informazioni sui contatti, visitare il sito Web [Microsoft Worldwide Information](http://www.microsoft.com/worldwide/), selezionare il Paese desiderato dall'elenco Informazioni sui contatti e fare clic su **Go** per visualizzare un elenco di numeri telefonici. Quando si effettua la chiamata, richiedere di parlare con il responsabile locale delle vendite per i servizi di supporto Premier. Vulnerabilità legata all'acquisizione di privilegi più elevati nelle password delle preferenze Criteri di gruppo - CVE-2014-1812
--------------------------------------------------------------------------------------------------------------------------------

<span id="sectionToggle3"></span>
Esiste una vulnerabilità legata all'acquisizione di privilegi più elevati nel modo in cui Active Directory distribuisce le password configurate tramite le preferenze Criteri di gruppo. Sfruttando questa vulnerabilità, un utente malintenzionato autenticato potrebbe decrittografare le password e utilizzarle al fine di acquisire privilegi più elevati per il dominio.

Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2014-1812](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2014-1812).

### Fattori attenuanti

Microsoft non ha individuato alcun fattore attenuante per questa vulnerabilità.

### Soluzioni alternative

Per soluzione alternativa si intende un'impostazione o una modifica nella configurazione che non elimina la vulnerabilità sottostante, ma consente di bloccare gli attacchi noti prima di applicare l'aggiornamento. Microsoft ha verificato le seguenti soluzioni alternative e segnala nel corso della discussione se tali soluzioni riducono la funzionalità:

-   Per informazioni sulla soluzione alternativa, vedere l'[articolo della Microsoft Knowledge Base 2962486](https://support.microsoft.com/kb/2962486).

### Domande frequenti

**Qual è la portata o l'impatto di questa vulnerabilità?**  
Si tratta di una vulnerabilità associata all'acquisizione di privilegi più elevati.

**Quali sono le cause di questa vulnerabilità?** 
La vulnerabilità è determinata dal modo in cui Active Directory distribuisce le password configurate tramite le preferenze Criteri di gruppo.

**Quali sono le preferenze Criteri di gruppo?** 
Le preferenze Criteri di gruppo comprendono oltre 20 nuove estensioni Criteri di gruppo che espandono la gamma delle impostazioni configurabili all'interno di un oggetto Criteri di gruppo (GPO). Queste nuove estensioni sono disponibili nella finestra Editor Gestione Criteri di gruppo nella Console Gestione Criteri di gruppo (GPMC), alla voce Preferenze. Le nuove estensioni delle preferenze Criteri di gruppo comprendono le impostazioni per opzioni di cartella, unità mappate, stampanti, attività pianificate, servizi e il menu Start. Per ulteriori informazioni sulle preferenze Criteri di gruppo, consultare la [Guida introduttiva alle preferenze Criteri di gruppo](http://technet.microsoft.com/library/cc731892).

**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?** 
Sfruttando questa vulnerabilità, un utente malintenzionato autenticato potrebbe acquisire le nuove credenziali di amministratore locale o di dominio e utilizzarle al fine di acquisire privilegi più elevati per il dominio. Questi è in grado di installare programmi, visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti amministrativi completi.

**In che modo un utente malintenzionato può sfruttare questa vulnerabilità?** 
Per sfruttare la vulnerabilità, un utente malintenzionato dovrebbe innanzitutto ottenere l'accesso a un account utente autenticato sul dominio. Se un oggetto Criteri di gruppo (GPO) è configurato tramite le preferenze Criteri di gruppo per impostare una password amministrativa locale o definire le credenziali per mappare un'unità di rete, pianificare un'attività o configurare il contesto di esecuzione di un servizio, l'utente malintenzionato può recuperare e decrittografare la password memorizzata con le preferenze Criteri di gruppo.

**Quali sono i sistemi principalmente interessati da questa vulnerabilità?** 
I server che funzionano come controller di dominio sono maggiormente esposti allo sfruttamento di questa vulnerabilità.

**Quali sono gli scopi dell'aggiornamento?** 
L'aggiornamento elimina la possibilità di configurare e distribuire le password utilizzando le seguenti estensioni di preferenze Criteri di gruppo.

-   Configurazione delle unità di mappatura
-   Configurazione di gruppi e utenti locali
-   Configurazione di attività pianificate
-   Configurazione di servizi
-   Configurazione di origini dati

**Altre operazioni richieste:** È importante tenere presente che l'aggiornamento non rimuove gli oggetti Criteri di gruppo (GPO) esistenti configurati prima dell'applicazione di questo aggiornamento per la protezione. I clienti con GPO esistenti, configurati utilizzando le preferenze Criteri di gruppo identificate, devono eliminare questo rischio dal proprio ambiente di dominio. Per ulteriori informazioni, consultare l'[articolo della Microsoft Knowledge Base 2962486](https://support.microsoft.com/kb/2962486).

**Al momento del rilascio di questo bollettino, le informazioni sulla vulnerabilità erano disponibili pubblicamente?** 
Sì. Le informazioni sulla vulnerabilità sono state divulgate pubblicamente. Questa vulnerabilità è stata identificata con il codice [CVE-2014-1812](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2014-1812).

**Al momento del rilascio di questo bollettino, erano già stati segnalati a Microsoft attacchi basati sullo sfruttamento di questa vulnerabilità?** 
Sì. Microsoft è a conoscenza di attacchi mirati che tentano di sfruttare questa vulnerabilità.

### Microsoft Active Protections Program (MAPP)
  
Per migliorare il livello di protezione offerto ai clienti, Microsoft fornisce ai principali fornitori di software di protezione i dati relativi alle vulnerabilità in anticipo rispetto alla pubblicazione mensile dell'aggiornamento per la protezione. I fornitori di software di protezione possono servirsi di tali dati per fornire ai clienti delle protezioni aggiornate tramite software o dispositivi di protezione, quali antivirus, sistemi di rilevamento delle intrusioni di rete o sistemi di prevenzione delle intrusioni basati su host. Per verificare se tali protezioni attive sono state rese disponibili dai fornitori di software di protezione, visitare i siti Web relativi alle protezioni attive pubblicati dai partner del programma, che sono elencati in [Microsoft Active Protections Program (MAPP) Partners](http://go.microsoft.com/fwlink/?linkid=215201).
