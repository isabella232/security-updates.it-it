Security Bulletin

Bollettino Microsoft sulla sicurezza MS10-010 - Importante
==========================================================

Una vulnerabilità in Windows Server 2008 Hyper-V può consentire un attacco di tipo Denial of Service (977894)
-------------------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 9 febbraio 2010

**Versione:** 1.0

### Informazioni generali

#### Riepilogo

Questo aggiornamento per la protezione risolve una vulnerabilità segnalata privatamente a Microsoft, che è stata riscontrata in Windows Server 2008 Hyper-V e in Windows Server 2008 R2 Hyper-V. Tale vulnerabilità può consentire un attacco di tipo Denial of Service se una sequenza non valida di istruzioni in linguaggio macchina viene eseguita da un utente autenticato in una delle macchine virtuali guest ospitate dal server Hyper-V. Per sfruttare tale vulnerabilità, un utente malintenzionato deve disporre di credenziali di accesso valide ed essere in grado di accedere in locale a una macchina virtuale guest. La vulnerabilità non può essere sfruttata in remoto o da utenti anonimi.

Questo aggiornamento per la protezione è considerato di livello importante per tutte le edizioni supportate basate su x64 di Windows Server 2008 e Windows Server 2008 R2. Per ulteriori informazioni, vedere la sottosezione **Software interessato e Software non interessato** all'interno della presente sezione.

L'aggiornamento per la protezione risolve la vulnerabilità correggendo il modo in cui il server Hyper-V convalida la codifica delle istruzioni in linguaggio macchina eseguite all'interno delle sue macchine virtuali guest. Per ulteriori informazioni sulla vulnerabilità, vedere la sottosezione Domande frequenti per la specifica voce della vulnerabilità nella sezione **Informazioni sulle vulnerabilità**.

**Raccomandazione.** Se la funzionalità Aggiornamenti automatici è abilitata, gli utenti non devono intraprendere alcuna azione, poiché questo aggiornamento per la protezione viene scaricato e installato automaticamente. Gli utenti che non hanno attivato la funzionalità Aggiornamenti automatici devono verificare la disponibilità di aggiornamenti e installare questo aggiornamento manualmente. Per informazioni sulle opzioni di configurazione specifiche relative agli aggiornamenti automatici, vedere l'[articolo della Microsoft Knowledge Base 294871](http://support.microsoft.com/kb/294871).

Per gli amministratori e le installazioni delle organizzazioni o gli utenti finali che desiderano installare manualmente questo aggiornamento per la protezione, Microsoft consiglia di applicare l'aggiornamento quanto prima utilizzando il software di gestione degli aggiornamenti o verificando la disponibilità degli aggiornamenti tramite il servizio [Microsoft Update](http://update.microsoft.com/microsoftupdate/v6/default.aspx?ln=it-it).

Vedere anche la sezione, **Strumenti e informazioni sul rilevamento e sulla distribuzione**, riportata di seguito nel presente bollettino.

**Che cos'è il server Hyper-V? È interessato anch'esso da questa vulnerabilità?**
Microsoft Hyper-V Server è un prodotto autonomo che offre una soluzione di virtualizzazione ottimizzata e affidabile che consente alle organizzazioni di migliorare l'utilizzo dei server e di ridurne i costi. Poiché Hyper-V Server è un prodotto dedicato autonomo, che contiene solamente l'hypervisor Windows, il WDM (Windows Driver Model) per il server e i componenti per la virtualizzazione, occupa una quantità limitata di memoria e assicura un sovraccarico minimo.
Microsoft Hyper-V Server 2008, Microsoft Hyper-V Server 2008 Service Pack 2 e Microsoft Hyper-V Server 2008 R2 sono interessati dalla vulnerabilità. Per questo motivo, per risolvere tale vulnerabilità, è necessario applicare rispettivamente gli aggiornamenti di Windows Server 2008 e Windows Server 2008 R2. Ai clienti che hanno abilitato Aggiornamenti automatici verrà offerto automaticamente di installare questi aggiornamenti.

Vulnerabilità legata alla convalida del set di istruzioni per Hyper-V - CVE-2010-0026  
-------------------------------------------------------------------------------------
  
<span></span>
In Windows Server 2008 Hyper-V e Windows Server 2008 R2 Hyper-V è stata riscontrata una vulnerabilità ad attacchi di tipo Denial of Service. Tale vulnerabilità è dovuta a una convalida insufficiente da parte di Hyper-V di sequenze di istruzioni specifiche in linguaggio macchina. Sfruttando la vulnerabilità, un utente malintenzionato può impedire al sistema Hyper-V interessato di rispondere alle richieste. Questo comportamento interessa tutte le macchine virtuali ospitate dal sistema.
  
Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2010-0026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0026).
  
#### Fattori attenuanti della vulnerabilità legata alla convalida del set di istruzioni per Hyper-V - CVE-2010-0026
  
Per fattore attenuante si intende un'impostazione, una configurazione comune o una procedura consigliata generica esistente in uno stato predefinito in grado di ridurre la gravità nello sfruttamento di una vulnerabilità. I seguenti fattori attenuanti possono essere utili per l'utente:
  
-   Per sfruttare tale vulnerabilità, un utente malintenzionato deve disporre di credenziali di accesso valide ed essere in grado di accedere in locale a una macchina virtuale Hyper-V. La vulnerabilità non può essere sfruttata in remoto o da utenti anonimi.
  
#### Soluzioni alternative per la vulnerabilità legata alla convalida del set di istruzioni per Hyper-V - CVE-2010-0026
  
Microsoft non ha individuato alcuna soluzione alternativa per questa vulnerabilità.
  
#### Domande frequenti sulla vulnerabilità legata alla convalida del set di istruzioni per Hyper-V - CVE-2010-0026
  
**Qual è la portata o l'impatto di questa vulnerabilità?**    
Si tratta di una vulnerabilità ad attacchi di tipo Denial of Service. Sfruttando questa vulnerabilità, un utente malintenzionato può impedire al server Hyper-V interessato di rispondere alle richieste e forzarne il riavvio. Si noti che la vulnerabilità ad attacchi di tipo Denial of Service non consente di eseguire codice o acquisire diritti utente più elevati, ma può impedire al sistema interessato di accettare richieste.
  
**Quali sono le cause di questa vulnerabilità?**    
La vulnerabilità è dovuta al fatto che il server Hyper-V convalida in modo errato la codifica di specifiche istruzioni in linguaggio macchina eseguite all'interno delle macchine virtuali guest. A causa di questa mancata convalida, l'elaborazione di tali istruzioni può provocare il blocco dell'applicazione server Hyper-V.
  
**Che cosa significa Hyper-V?**    
Hyper-V è una tecnologia basata su hypervisor che rappresenta una funzionalità chiave di Windows Server 2008 e Windows Server 2008 R2. Fornisce una piattaforma di virtualizzazione scalabile, affidabile e ad elevata disponibilità. Un ambiente di virtualizzazione consente agli utenti di creare e gestire macchine virtuali, eseguendo più sistemi operativi in un unico computer fisico e isolando i sistemi l'uno dall'altro.
  
**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**    
Sfruttando questa vulnerabilità, un utente malintenzionato può provocare il blocco del sistema e forzarne il riavvio. È importante notare che lo sfruttamento di questa vulnerabilità può provocare il blocco non soltanto del server Hyper-V effettivo ma anche di tutte le macchine virtuali guest in esso ospitate.
  
**In quale modo un utente malintenzionato può sfruttare questa vulnerabilità?**    
Un utente malintenzionato deve venire autenticato da una delle macchine virtuali guest ospitate dal server Hyper-V ed essere in grado di eseguire codice arbitrario all'interno del sistema. L'utente malintenzionato può quindi eseguire un file eseguibile non attendibile nel sistema che richiami una sequenza non valida di istruzioni in linguaggio macchina in grado di provocare il blocco del server Hyper-V.
  
**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**    
Sono particolarmente esposti a questa vulnerabilità i server Hyper-V che ospitano macchine virtuali guest che consentono l'accesso a utenti non attendibili.
  
**Quali sono gli scopi dell'aggiornamento?**    
L'aggiornamento risolve la vulnerabilità correggendo il modo in cui il server Hyper-V convalida la codifica delle istruzioni in linguaggio macchina eseguite all'interno delle sue macchine virtuali guest.
  
**Al momento del rilascio di questo bollettino le informazioni sulla vulnerabilità erano disponibili pubblicamente?**    
No. Microsoft ha ricevuto informazioni sulla vulnerabilità da fonti private.
  
**Al momento del rilascio di questo bollettino erano già stati segnalati attacchi basati sullo sfruttamento di questa vulnerabilità?**    
No. Al momento della pubblicazione del presente bollettino, Microsoft non aveva ricevuto alcuna segnalazione in merito allo sfruttamento di questa vulnerabilità a scopo di attacco.
