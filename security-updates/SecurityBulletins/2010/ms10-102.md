Security Bulletin

Bollettino Microsoft sulla sicurezza MS10-102 - Importante
==========================================================

Una vulnerabilità presente in Hyper-V può consentire un attacco di tipo Denial of Service (2345316)
---------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 14 dicembre 2010 | Aggiornamento: mercoledì 12 gennaio 2011

**Versione:** 1.1

### Informazioni generali

#### Riepilogo

Questo aggiornamento per la protezione risolve una vulnerabilità segnalata privatamente a Microsoft, che è stata riscontrata in Windows Server 2008 Hyper-V e in Windows Server 2008 R2 Hyper-V. La vulnerabilità può consentire un attacco di tipo Denial of Service se un pacchetto appositamente predisposto viene inviato al VMBus da un utente autenticato in una delle macchine virtuali guest ospitate dal server Hyper-V. Per sfruttare la vulnerabilità, un utente malintenzionato deve essere in possesso di credenziali di accesso valide ed essere in grado di inviare contenuto appositamente predisposto da una macchina virtuale guest. La vulnerabilità non può essere sfruttata in remoto o da utenti anonimi.

Questo aggiornamento per la protezione è considerato di livello importante per tutte le edizioni supportate basate su x64 di Windows Server 2008 e Windows Server 2008 R2. Per ulteriori informazioni, vedere la sottosezione **Software interessato e Software non interessato** all'interno della presente sezione.

L'aggiornamento per la protezione risolve la vulnerabilità correggendo il modo in cui il server Hyper-V convalida i pacchetti non validi inviati al VMBus all'interno delle relative macchine virtuali guest. Per ulteriori informazioni sulla vulnerabilità, vedere la sottosezione Domande frequenti per la specifica voce della vulnerabilità nella sezione **Informazioni sulle vulnerabilità**.

**Raccomandazione.** Se la funzionalità Aggiornamenti automatici è abilitata, gli utenti non devono intraprendere alcuna azione, poiché questo aggiornamento per la protezione viene scaricato e installato automaticamente. Gli utenti che non hanno attivato la funzionalità Aggiornamenti automatici devono verificare la disponibilità di aggiornamenti e installare questo aggiornamento manualmente. Per informazioni sulle opzioni di configurazione specifiche relative agli aggiornamenti automatici, vedere l'[articolo della Microsoft Knowledge Base 294871](http://support.microsoft.com/kb/294871).

Per gli amministratori e le installazioni delle organizzazioni o gli utenti finali che desiderano installare manualmente questo aggiornamento per la protezione, Microsoft consiglia di applicare l'aggiornamento quanto prima utilizzando il software di gestione degli aggiornamenti o verificando la disponibilità degli aggiornamenti tramite il servizio [Microsoft Update](http://www.update.microsoft.com/microsoftupdate/v6/vistadefault.aspx?ln=it-it).

Vedere anche la sezione, **Strumenti e informazioni sul rilevamento e sulla distribuzione**, riportata di seguito nel presente bollettino.

**Problemi noti.**  L'[articolo della Microsoft Knowledge Base 2345316](http://support.microsoft.com/kb/2345316) descrive i problemi attualmente conosciuti che gli utenti potrebbero riscontrare durante l'installazione di questo aggiornamento per la protezione. L'articolo illustra inoltre le soluzioni consigliate in grado di risolvere questi problemi.

Domande frequenti sull'aggiornamento per la protezione
------------------------------------------------------

<span></span>
**Il ruolo di Hyper-V non è attivato. È possibile installare questo aggiornamento?** 
No. L'aggiornamento può essere installato solo sui sistemi operativi interessati con il ruolo di Hyper-V attivato. I sistemi con il ruolo di Hyper-V disattivato non sono interessati dalla vulnerabilità.

**Questo aggiornamento è disponibile anche se Hyper-V non è stato attivato?**  
L'aggiornamento viene offerto tramite Aggiornamenti automatici solamente se l'aggiornamento automatico del ruolo Hyper-V è stato attivato. Ciò vale a prescindere dal fatto che il sistema operativo interessato sia stato installato o meno mediante l'opzione di installazione Server Core.

**Questo aggiornamento deve essere applicato alle macchine virtuali guest di Hyper-V?**  
Questo aggiornamento non deve essere applicato alle macchine guest di Hyper-V ma solamente ai server che hanno attivato il ruolo Hyper-V. Viene offerto tramite Aggiornamenti automatici soltanto al sistema server Hyper-V e non alle macchine guest virtualizzate.

**Che cos'è il server Hyper-V? È interessato anch'esso da questa vulnerabilità?**  
Microsoft Hyper-V Server è un prodotto autonomo che offre una soluzione di virtualizzazione ottimizzata e affidabile che consente alle organizzazioni di migliorare l'utilizzo dei server e di ridurne i costi. Poiché Hyper-V Server è un prodotto dedicato autonomo, che contiene solamente l'hypervisor Windows, il WDM (Windows Driver Model) per il server e i componenti per la virtualizzazione, occupa una quantità limitata di memoria e assicura un sovraccarico minimo.

Microsoft Hyper-V Server 2008, Microsoft Hyper-V Server 2008 Service Pack 2 e Microsoft Hyper-V Server 2008 R2 sono interessati da questa vulnerabilità. I rispettivi aggiornamenti di Windows Server 2008 e Windows Server 2008 R2 devono essere applicati per risolvere questa vulnerabilità. Ai clienti che hanno abilitato Aggiornamenti automatici verrà offerto automaticamente di installare questi aggiornamenti.

**Quali sono i problemi noti che potrebbero verificarsi quando si disinstalla questo aggiornamento per la protezione?**   
La disinstallazione di questo aggiornamento richiede due riavvii a causa della presenza di determinati file binari nell'aggiornamento. I sistemi che non vengono riavviati due volte dopo la disinstallazione dell'aggiornamento potrebbe riscontrare una perdita di connettività della rete. Per ulteriori informazioni su questo problema, consultare l'[articolo della Microsoft Knowledge Base 2345316](http://support.microsoft.com/kb/2345316).

Vulnerabilità VMBus in Hyper-V - CVE-2010-3960  
----------------------------------------------
  
<span></span>
In Windows Server 2008 Hyper-V e Windows Server 2008 R2 Hyper-V è stata riscontrata una vulnerabilità che può consentire un attacco di tipo Denial of Service se un pacchetto specificamente predisposto viene inviato al VMBus da un utente autenticato in una delle macchine virtuali guest ospitata dal server Hyper-V. Per sfruttare tale vulnerabilità, un utente malintenzionato deve disporre di credenziali di accesso valide ed essere in grado di accedere in locale a una macchina virtuale guest. La vulnerabilità non può essere sfruttata in remoto o da utenti anonimi.
  
Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2010-3960](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3960).
  
#### Fattori attenuanti della vulnerabilità VMBus in Hyper-V - CVE-2010-3960
  
Per fattore attenuante si intende un'impostazione, una configurazione comune o una procedura consigliata generica esistente in uno stato predefinito in grado di ridurre la gravità nello sfruttamento di una vulnerabilità. I seguenti fattori attenuanti possono essere utili per l'utente:
  
-   Per sfruttare tale vulnerabilità, un utente malintenzionato deve disporre di credenziali di accesso valide ed essere in grado di accedere in locale a una macchina virtuale Hyper-V. La vulnerabilità non può essere sfruttata in remoto o da utenti anonimi.
  
#### Soluzioni alternative per la vulnerabilità VMBus in Hyper-V - CVE-2010-3960
  
Microsoft non ha individuato alcuna soluzione alternativa per questa vulnerabilità.
  
#### Domande frequenti sulla vulnerabilità VMBus in Hyper-V - CVE-2010-3960
  
**Qual è la portata o l'impatto di questa vulnerabilità?**    
Si tratta di una vulnerabilità ad attacchi di tipo Denial of Service. Sfruttando questa vulnerabilità, un utente malintenzionato può impedire al server Hyper-V interessato di rispondere alle richieste e forzarne il riavvio. Si noti che la vulnerabilità ad attacchi di tipo Denial of Service non consente di eseguire codice o acquisire diritti utente più elevati, ma può impedire al sistema interessato di accettare richieste.
  
**Quali sono le cause di questa vulnerabilità?**    
Questa vulnerabilità si verifica quando il server Hyper-V convalida in maniera errata i pacchetti appositamente predisposti inviati al VMBus in Hyper-V. A causa di questa mancata convalida, l'elaborazione di tali pacchetti può provocare il blocco dell'applicazione server Hyper-V.
  
**Che cos'è Hyper-V?**    
Hyper-V è una tecnologia basata su hypervisor che rappresenta una funzionalità chiave di Windows Server 2008 e Windows Server 2008 R2. Fornisce una piattaforma di virtualizzazione scalabile, affidabile e ad elevata disponibilità. Un ambiente di virtualizzazione consente agli utenti di creare e gestire macchine virtuali, eseguendo più sistemi operativi in un unico computer fisico e isolando i sistemi l'uno dall'altro.
  
**Che cos'è il VMBus?**    
VMBus è un canale di comunicazione ad interpartizione logica. Per ulteriori informazioni sull'architettura VMBus e Hyper-V, vedere l'articolo MSDN [Architettura di Hyper-V](http://msdn.microsoft.com/en-us/library/cc768520(bts.10).aspx).
  
**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**    
Sfruttando questa vulnerabilità, un utente malintenzionato può provocare il blocco del sistema e forzarne il riavvio. È importante notare che lo sfruttamento di questa vulnerabilità può provocare il blocco non soltanto del server Hyper-V effettivo ma anche di tutte le macchine virtuali guest in esso ospitate.
  
**In che modo un utente malintenzionato può sfruttare questa vulnerabilità?**    
Un utente malintenzionato deve venire autenticato da una delle macchine virtuali guest ospitate dal server Hyper-V ed essere in grado di eseguire codice arbitrario all'interno del sistema. Un utente malintenzionato deve quindi creare un driver personalizzato che crea pacchetti appositamente predisposti da inoltrare al VMBus causando così il blocco del server Hyper-V.
  
**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**    
Sono particolarmente esposti a questa vulnerabilità i server Hyper-V che ospitano macchine virtuali guest che consentono l'accesso a utenti non attendibili.
  
**Quali sono gli scopi dell'aggiornamento?**    
L'aggiornamento risolve la vulnerabilità correggendo il modo in cui il server Hyper-V convalida e gestisce i pacchetti appositamente predisposti inoltrati dalle macchine virtuali guest al VMBus in Hyper-V.
  
**Al momento del rilascio di questo bollettino le informazioni sulla vulnerabilità erano disponibili pubblicamente?**    
No. Microsoft ha ricevuto informazioni sulla vulnerabilità grazie ad un'indagine coordinata.
  
**Al momento del rilascio di questo bollettino erano già stati segnalati attacchi basati sullo sfruttamento di questa vulnerabilità?**    
No. Al momento della pubblicazione del presente bollettino, Microsoft non aveva ricevuto alcuna segnalazione in merito allo sfruttamento di questa vulnerabilità a scopo di attacco.
