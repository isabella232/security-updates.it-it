Security Bulletin

Bollettino Microsoft sulla sicurezza MS07-022 - Importante
==========================================================

Una vulnerabilità nel kernel di Windows può consentire l'acquisizione di privilegi più elevati (931784)
-------------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 10 aprile 2007 | Aggiornamento: martedì 26 giugno 2007

**Versione:** 2.0

#### Riepilogo

**Destinatari:** Clienti che utilizzano Microsoft Windows

**Effetti della vulnerabilità:** Acquisizione di privilegi più elevati

**Livello di gravità:** Importante

**Raccomandazione:** È consigliabile installare l'aggiornamento per la protezione il prima possibile

**Aggiornamenti per la protezione sostituiti:** Questo bollettino sostituisce un aggiornamento per la protezione precedente. Per ulteriori informazioni, vedere la sezione Domande frequenti del presente bollettino.

**Software verificato e posizioni per il download dell'aggiornamento per la protezione:**

**Software interessato:**

-   Microsoft Windows 2000 Service Pack 4 - [Download dell'aggiornamento](http://www.microsoft.com/downloads/details.aspx?displaylang=it&familyid=b3599afb-7673-4ef6-a2b1-d77e39fd782c)
-   Microsoft Windows XP Service Pack 2 - [Download dell'aggiornamento](http://www.microsoft.com/downloads/details.aspx?displaylang=it&familyid=eeaee4a7-4858-4b6b-9c6d-a9f1eae19b51)
-   Microsoft Windows Server 2003, Microsoft Windows Server 2003 Service Pack 1 e Microsoft 2003 Service Pack 2 - [Download dell'aggiornamento](http://www.microsoft.com/downloads/details.aspx?displaylang=it&familyid=5e94a29c-7ec0-4459-92eb-d43b524fd6fd)

-   Microsoft Windows Server 2003 per sistemi basati su Itanium e Microsoft Windows Server 2003 con SP1 e SP2 per sistemi basati su Itanium

-   Windows Vista x64 Edition

Domande frequenti sull'aggiornamento per la protezione

**Perché Microsoft ha aggiornato questo bollettino in data 26 giugno 2007?**
Microsoft ha aggiornato questo bollettino per avvertire i clienti con Windows 2000 Service Pack 4 installato in sistemi NEC 98 della disponibilità della versione rivista dell'aggiornamento per la protezione. Potrebbero verificarsi problemi in questi sistemi durante i tentativi di connessione a Windows Update dovuti a una versione della build non corretta dell'aggiornamento per la protezione per i sistemi NEC 98. L'aggiornamento per la protezione originario risolveva con successo le vulnerabilità descritte in questo bollettino di protezione.

Microsoft consiglia ai clienti di NEC 98 con Windows 2000 Service Pack 4 di scaricare l'aggiornamento per la protezione rivisto dal sito Web Area download Microsoft per assicurare che i download futuri di aggiornamenti per la protezione da Windows Update siano corretti. I clienti di NEC 98 che eseguono Windows 2000 Service Pack 4 potrebbero anche disinstallare questo aggiornamento per la protezione per ricevere automaticamente l'aggiornamento rivisto da Windows Update.

Per ulteriori informazioni su questo problema, vedere l' [articolo della Microsoft Knowledge Base](http://support.microsoft.com/kb/931784/)[931784](http://support.microsoft.com/kb/931784/) che descrive i problemi attualmente conosciuti che potrebbero verificarsi quando si installa questo aggiornamento per la protezione.

Dettagli della vulnerabilità
----------------------------

<span></span>
#### Vulnerabilità legata all'acquisizione locale di privilegi più elevati nel kernel - CVE-2007-1206:

Nel kernel di Windows esiste una vulnerabilità legata all'acquisizione di privilegi più elevati dovuta ad autorizzazioni non corrette su un segmento di memoria con mapping. Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe assumere il pieno controllo del sistema interessato,

#### Fattori attenuanti della vulnerabilità legata all'acquisizione locale di privilegi più elevati nel kernel - CVE-2007-1206:

Per sfruttare la vulnerabilità, è necessario disporre di credenziali di accesso valide ed essere in grado di accedere al sistema in locale. La vulnerabilità non può essere sfruttata in remoto o da utenti anonimi.

#### Soluzioni alternative per la vulnerabilità legata all'acquisizione locale di privilegi più elevati nel kernel - CVE-2007-1206:

Non è stata individuata alcuna soluzione alternativa per questa vulnerabilità.

#### Domande frequenti sulla vulnerabilità legata all'acquisizione locale di privilegi più elevati nel kernel - CVE-2007-1206:

**Perché questo bollettino viene nuovamente proposto? **
I clienti che eseguono Windows 2000 in computer NEC 98 potrebbero non aver ricevuto correttamente l'aggiornamento per la protezione 931784. Il bollettino è stato rilasciato nuovamente per risolvere questo problema. Questa nuova versione del bollettino è solo per i clienti che utilizzano computer NEC 98; gli altri clienti non sono interessati.

**Qual è la portata o l'impatto di questa vulnerabilità? **
Nel kernel di Windows esiste una vulnerabilità legata all'acquisizione di privilegi più elevati dovuta ad autorizzazioni non corrette su un segmento di memoria con mapping. Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe assumere il pieno controllo del sistema interessato, Per tentare di sfruttare la vulnerabilità, è necessario accedere al sistema in locale.

**Quali sono le cause di questa vulnerabilità? **
Il kernel di Windows consente di utilizzare autorizzazioni non corrette durante il mapping di un segmento di memoria.

**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità? **
Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe eseguire codice dannoso nel sistema di un utente con privilegi elevati. In questo modo un utente malintenzionato potrebbe assumere il controllo completo di un sistema interessato.

**In quale modo un utente malintenzionato può sfruttare questa vulnerabilità? **
Per sfruttare la vulnerabilità, è necessario che un utente malintenzionato acceda al sistema quindi occorre eseguire un'applicazione appositamente predisposta e assumere il controllo completo del sistema.

**Quali sono i sistemi principalmente interessati da questa vulnerabilità? **
I sistemi maggiormente a rischio sono workstation e server terminal. I server potrebbero essere maggiormente a rischio se gli amministratori consentono agli utenti di accedere ai server ed eseguire programmi. Tuttavia, le procedure consigliate scoraggiano fortemente questa attribuzione di privilegi.

**È possibile sfruttare questa vulnerabilità tramite Internet? **
No. L'utente malintenzionato deve accedere al sistema specifico a cui è destinato l'attacco e non può caricare ed eseguire un programma in remoto sfruttando la vulnerabilità.

**Quali sono gli scopi dell'aggiornamento? **
L'aggiornamento rimuove la vulnerabilità modificando il modo in cui il kernel di Windows convalida le autorizzazioni durante il mapping della memoria.

**Al momento del rilascio di questo bollettino le informazioni sulla vulnerabilità erano disponibili pubblicamente? **
No. Microsoft ha ricevuto informazioni sulla vulnerabilità da fonti private. Secondo le informazioni in possesso di Microsoft, al momento della pubblicazione del presente bollettino l'esistenza di questa vulnerabilità non era stata ancora divulgata pubblicamente.

**Al momento del rilascio di questo bollettino erano già stati segnalati attacchi basati sullo sfruttamento di questa vulnerabilità? **
No. Al momento della pubblicazione del presente bollettino, Microsoft non aveva ricevuto alcuna segnalazione in merito allo sfruttamento di questa vulnerabilità a scopo di attacco, né dell'esistenza di un codice di prova pubblicato.
