Security Bulletin

Bollettino Microsoft sulla sicurezza MS08-028 - Critico
=======================================================

Una vulnerabilità nel modulo di gestione di database Microsoft Jet può consentire l'esecuzione di codice (950749)
-----------------------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 13 maggio 2008 | Aggiornamento: mercoledì 16 luglio 2008

**Versione:** 1.3

### Informazioni generali

#### Riepilogo

Questo aggiornamento per la protezione risolve una vulnerabilità nel modulo di gestione di database Microsoft Jet (Jet) in Windows. Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe assumere il pieno controllo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi. Gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.

Questo è un aggiornamento per la protezione di livello critico per il modulo di gestione di database Microsoft Jet 4.0. Per ulteriori informazioni, vedere la sottosezione **Software interessato e Software non interessato** in questa sezione.

Questo aggiornamento per la protezione risolve la vulnerabilità modificando il modo in cui il modulo di gestione di database Microsoft Jet analizza i dati in un database. Per ulteriori informazioni sulla vulnerabilità, vedere la sottosezione Domande frequenti per la specifica voce della vulnerabilità nella sezione **Informazioni sulle vulnerabilità**.

Questo aggiornamento per la protezione risolve anche la vulnerabilità descritta nell'[Advisory Microsoft sulla sicurezza 950627](http://technet.microsoft.com/security/advisory/950627). Oltre a installare questo aggiornamento, si consiglia ai clienti con Microsoft Word di installare gli aggiornamenti forniti nel [Bollettino Microsoft sulla sicurezza MS08-026: Alcune vulnerabilità in Microsoft Word possono consentire l'esecuzione di codice in modalità remota (951207)](http://technet.microsoft.com/security/bulletin/ms08-026) per avere una protezione aggiornata da attacchi di questo tipo.

**Raccomandazione.** Microsoft consiglia di applicare l'aggiornamento immediatamente.

**Problemi noti.** Nessuno

**Anche CVE-2005-0944 descrive una vulnerabilità nel modulo di gestione di database Microsoft Jet. Qual è la relazione tra CVE-2005-0944 e questo bollettino?**
In questo aggiornamento viene affrontata anche la vulnerabilità indicata in CVE-2005-0944. **Perché questo aggiornamento è di livello critico?**
Benché il file mdb sia nell'elenco dei tipi di file dannosi, esistono scenari di attacco in grado di eludere i fattori attenuanti per i tipi di file dannosi. In particolare, i clienti con Microsoft Office potrebbero essere esposti in scenari di attacchi basati su posta elettronica o download diretti. Ad esempio, l'utente malintenzionato potrebbe sfruttare la vulnerabilità inviando un file Word con un file mdb appositamente predisposto incorporato all'utente e convincendolo ad aprire il documento o a visualizzare il messaggio di posta elettronica contenente il file appositamente predisposto. Nel caso di Outlook 2003 e Outlook 2007, l'attacco potrebbe essere eseguito facendo aprire il documento in HTML nel riquadro di anteprima di Outlook 2003 o Outlook 2007. Microsoft consiglia di applicare l'aggiornamento immediatamente.

**Questo aggiornamento fornisce protezione da attacchi tramite altre applicazioni?**
Sì. Esistono alcune applicazioni che possono essere installate e utilizzate come vettore di attacco per questa vulnerabilità. Ad esempio, un file di database Jet può essere aperto da un documento Word, un messaggio di posta elettronica o visitando un sito Web dannoso. Applicando questo aggiornamento, è possibile bloccare gli attacchi correnti. Tuttavia, si consiglia ai clienti di applicare anche gli aggiornamenti forniti nel [Bollettino Microsoft sulla sicurezza MS08-026: Alcune vulnerabilità in Microsoft Word possono consentire l'esecuzione di codice in modalità remota (951207)](http://technet.microsoft.com/security/bulletin/ms08-026) per bloccare completamente i nuovi vettori di attacco.

**Esistono applicazioni e servizi che utilizzano Extensible Storage Engine (ESE), noto in passato come JET Blue?**
No. Questa vulnerabilità si applica soltanto al modulo di gestione di database Microsoft Jet 4.0. Le applicazioni e i servizi che utilizzano Extensible Storage Engine non sono interessati.

Vulnerabilità legata all'overflow dello stack di analisi dei file MDB del modulo di gestione di database Microsoft Jet - CVE-2007-6026  
--------------------------------------------------------------------------------------------------------------------------------------
  
<span></span>
Una vulnerabilità di buffer overrun presente nel motore di database Microsoft Jet può consentire l'esecuzione di codice in modalità remota in un sistema interessato. Un utente malintenzionato può sfruttare questa vulnerabilità creando una query di database appositamente formulata e inviandola a un'applicazione che utilizza Jet in un sistema interessato. Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe assumere il pieno controllo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi. Gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.
  
Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2007-6026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-6026).
  
#### Fattori attenuanti della vulnerabilità legata all'overflow dello stack di analisi dei file MDB del modulo di gestione di database Microsoft Jet - CVE-2007-6026
  
Per fattore attenuante si intende un'impostazione, una configurazione comune o una procedura consigliata generica esistente in uno stato predefinito in grado di ridurre la gravità nello sfruttamento di una vulnerabilità. I seguenti fattori attenuanti possono essere utili per l'utente:
  
-   Sfruttando questa vulnerabilità, un utente malintenzionato può ottenere gli stessi diritti utente dell'utente locale. Pertanto, gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.  
-   In uno scenario di attacco dal Web, l'utente malintenzionato dovrebbe pubblicare un sito contenente un file Word appositamente predisposto utilizzato per tentare di sfruttare questa vulnerabilità. Inoltre, i siti Web manomessi e quelli che accettano o ospitano contenuti forniti dagli utenti potrebbero presentare contenuti appositamente predisposti per sfruttare questa vulnerabilità. Poiché non è in alcun modo possibile obbligare gli utenti a visitare questi siti Web, l'utente malintenzionato dovrebbe invogliare le vittime a visitare il sito Web, in genere inducendole a fare clic su un collegamento in un messaggio di posta elettronica o di Instant Messenger che le indirizzi al sito.  
-   I sistemi che utilizzano Microsoft Outlook 2003 e 2007 come client di posta elettronica possono attenuare il vettore di posta elettronica HTML per Outlook 2007 configurando la lettura dei messaggi solo in testo normale.  
-   I sistemi che eseguono tutte le versioni supportate di Windows XP Service Pack 3, Windows Server 2003 Service Pack 2, Windows Vista e Windows Server 2008 non sono interessati da questa vulnerabilità.
  
#### Soluzioni alternative per la vulnerabilità legata all'overflow dello stack di analisi dei file MDB del modulo di gestione di database Microsoft Jet - CVE-2007-6026
  
Per soluzione alternativa si intende un'impostazione o una modifica nella configurazione che non elimina la vulnerabilità sottostante, ma consente di bloccare gli attacchi noti prima di applicare l'aggiornamento. Microsoft ha verificato le seguenti soluzioni alternative e segnala nel corso della discussione se tali soluzioni riducono la funzionalità:
  
-   **Impedire al modulo di gestione di database Microsoft Jet di essere eseguito per qualsiasi applicazione**
  
    Per implementare la soluzione alternativa, al prompt dei comandi digitare quanto segue:
  
    `echo y| cacls "%SystemRoot%\system32\msjet40.dll" /E /P everyone:N`
  
    **Effetto della soluzione alternativa.** Qualsiasi applicazione che richieda l'utilizzo del modulo di gestione di database Microsoft Jet per le chiamate di accesso ai dati non funzionerà.
  
    **Per annullare il risultato della soluzione alternativa.** Al prompt dei comandi, inserire quanto segue
  
    `echo y| cacls "%SystemRoot%\system32\msjet40.dll" /E /R everyone`
  
-   **Utilizzare Criteri di gruppo per impedire al modulo di gestione di database Microsoft Jet di essere eseguito per qualsiasi applicazione**
  
    Per implementare la soluzione alternativa, eseguire i passaggi seguenti:
  
    1.  Creare il seguente script, denominato JetCacls.cmd:  
        @echo off  
        if exist %systemdrive%\\Cacls.log goto end  
        cacls "%SystemRoot%\\system32\\msjet40.dll" /E /P everyone:N &gt; nul 2&gt;&1  
        echo %date% %time%: Msjet Cacls updated &gt; %systemdrive%\\Cacls.log  
        :end  
        exit  
    2.  Copiare JetCacls.cmd nella cartella condivisa Netlogon o in un'altra cartella condivisa sul controller di dominio dal quale viene eseguito JetCacls.cmd.  
    3.  Configurare JetCacls.cmd. Nello snap-in **MMC per i computer e gli utenti di Active Directory**, fare clic con il pulsante destro del mouse sul nome dominio e scegliere **Proprietà**.  
    4.  Fare clic sulla scheda **Criteri di gruppo**.  
    5.  Scegliere **Nuovo** per creare un nuovo oggetto Criteri di gruppo (GPO) e inserire JetCacls come nome del criterio.  
    6.  Fare clic sul nuovo criterio e scegliere **Modifica**.  
    7.  Espandere le **impostazioni Windows per la configurazione del computer** e scegliere **Script**.  
    8.  fare doppio clic su **Accesso** e scegliere **Aggiungi**. Viene visualizzata la finestra di dialogo Aggiunta script.  
    9.  Digitare \\\\s*ervername*\\s*harename*\\JetCacls.cmd nella casella Nome script.  
    10. Scegliere **OK** e quindi **Applica**.  
    11. Riavviare i computer client membri di questo dominio.
  
    **Effetto della soluzione alternativa.** Qualsiasi applicazione che richieda l'utilizzo del modulo di gestione di database Microsoft Jet per le chiamate di accesso ai dati non funzionerà. Questa restrizione si applica solo alle applicazioni eseguite sui computer client del dominio.
  
-   **Blocco dell'esecuzione dei file MDB dall'infrastruttura di posta elettronica**
  
    **Nota** Tutti i file di database Jet devono essere elaborati come tipi di file non sicuri per gli utenti comuni. Inoltre, Microsoft consiglia di considerare come sospetti anche i file di database trasferiti tramite posta elettronica.
  
    Per implementare questa soluzione alternativa, l'ambiente di posta elettronica deve supportare la funzionalità di ricerca degli allegati che contengono una specifica struttura di file (non solo l'estensione di file) all'interno di un singolo messaggio di posta elettronica. È poi possibile intraprendere azioni sull'allegato come l'eliminazione, la quarantena, la notifica e la segnalazione del file individuato.
  
    Per rilevare file Jet ai quali sia stato modificato il nome in un altro tipo di file, cercare tutti i file con una delle seguenti firme a 15 byte nella posizione 0x4 (senza virgolette):
  
    `"Jet System DB  ""Standard Jet DB""Temp Jet DB    "`
  
    Per le configurazioni specifiche dei clienti di Microsoft Exchange che utilizzano le tecnologie Forefront (in passato Antigen), vedere [Microsoft Forefront Server Security: File Filtering](http://technet.microsoft.com/it-it/library/bb795068.aspx) (in inglese) per ulteriori informazioni.
  
    Queste informazioni sono state condivise con i membri di [Microsoft Security Response Alliance](http://www.microsoft.com/security/msra/default.mspx). Per utilizzare gli strumenti MSRA per rilevare i file MDB, contattare i fornitori presenti nell'elenco sulla home page di MSRA.
  
    **Impatto della soluzione alternativa.** L'esecuzione dei file rilevati da questa configurazione sarà bloccata tramite il sistema di posta elettronica dell'organizzazione.
  
-   Configurare Outlook 2007 in modo da leggere i messaggi in testo normale.  
-   Non aprire o salvare file Microsoft Word o Jet provenienti da fonti non attendibili oppure ricevuti inaspettatamente da fonti attendibili. Questa vulnerabilità può essere sfruttata quando un utente apre un file appositamente predisposto.
  
#### Domande frequenti sulla vulnerabilità legata all'overflow dello stack di analisi dei file MDB del modulo di gestione di database Microsoft Jet - CVE-2007-6026
  
**Qual è la portata o l'impatto di questa vulnerabilità?**  
Si tratta di una vulnerabilità legata all'esecuzione di codice in modalità remota. Sfruttando questa vulnerabilità, un utente malintenzionato anonimo remoto potrebbe eseguire codice non autorizzato nel contesto dell'utente locale.
  
**Quali sono le cause di questa vulnerabilità?**  
Il modulo di gestione di database Microsoft Jet (Jet) non esegue una convalida adeguata della struttura di dati. Un utente malintenzionato può sfruttare questa vulnerabilità creando una query di database appositamente formulata e inviandola a un'applicazione che utilizza Jet in un sistema interessato.
  
**Che cos'è il modulo di gestione di database Microsoft Jet?**  
Il modulo di gestione di database Microsoft Jet rende possibile l'accesso ai dati da varie applicazioni, ad esempio Microsoft Access, Microsoft Visual Basic e numerose applicazioni di terze parti. Jet può inoltre essere utilizzato da applicazioni Internet che necessitano della funzionalità di database.
  
**I file mdb non si trovano nell'elenco di tipi di file dannosi? In questo caso, perché questa vulnerabilità è di livello critico?**  
Mentre i file mdb vengono elencati tra i tipi di file dannosi, è stato rilevato un nuovo vettore di attacco quando un file di database Jet può essere aperto da un documento Word. Questo aggiornamento bloccherà gli attacchi correnti oltre a fornire ulteriori correzioni per il modulo di gestione di database Jet. Tuttavia, si consiglia ai clienti di applicare anche gli aggiornamenti forniti nel [Bollettino Microsoft sulla sicurezza MS08-026: Alcune vulnerabilità in Microsoft Word possono consentire l'esecuzione di codice in modalità remota (951207)](http://technet.microsoft.com/security/bulletin/ms08-026) per bloccare il nuovo vettore di attacco.
  
**I file mdb devono essere ancora considerati dannosi?**  
Sì. Mentre questo aggiornamento blocca gli attacchi attivi, i file di database mdb eseguono funzioni dannose. I clienti dovrebbero utilizzare questi file con cautela. Inoltre, le applicazioni di posta elettronica Microsoft supportate continueranno a bloccare l'apertura dei file mdb.
  
**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**  
Sfruttando questa vulnerabilità, un utente malintenzionato può ottenere gli stessi diritti utente dell'utente connesso. Se un utente è connesso con privilegi di amministrazione, un utente malintenzionato può assumere il controllo completo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account con diritti utente completi. Gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.
  
**In quale modo un utente malintenzionato può sfruttare questa vulnerabilità?**  
Un utente malintenzionato può sfruttare questa vulnerabilità convincendo un utente ad aprire un file di database appositamente predisposto utilizzando Jet. Sfruttando questa vulnerabilità, un utente malintenzionato può ottenere gli stessi diritti utente dell'utente locale. Pertanto, gli utenti con account configurati in modo da disporre solo di diritti limitati sono esposti all'attacco in misura inferiore rispetto a quelli che operano con privilegi di amministrazione.
  
In uno scenario di attacco di posta elettronica, l'utente malintenzionato potrebbe sfruttare la vulnerabilità inviando un file Word con un file mdb appositamente predisposto incorporato all'utente e convincendolo ad aprire il documento o il messaggio di posta elettronica contenente il file appositamente predisposto. Nel caso di Outlook 2003 e Outlook 2007, invece, l'attacco potrebbe essere eseguito facendo aprire il documento in HTML nel riquadro di anteprima di Outlook 2003 o Outlook 2007.
  
In uno scenario di attacco dal Web, l'utente malintenzionato dovrebbe pubblicare un sito Web contenente un file Jet o un file Word con un file Jet appositamente predisposto incorporato per tentare di sfruttare questa vulnerabilità. Inoltre, i siti Web manomessi e quelli che accettano o ospitano contenuti forniti dagli utenti potrebbero presentare contenuti appositamente predisposti per sfruttare questa vulnerabilità. Poiché non è in alcun modo possibile obbligare gli utenti a visitare un sito Web appositamente predisposto, L'utente malintenzionato dovrebbe invece invogliare le vittime a visitare il sito, in genere inducendole a fare clic su un collegamento in un messaggio di posta elettronica o di Instant Messenger che le indirizzi al sito.
  
**È possibile ridurre il rischio di un attacco tramite posta elettronica HTML?**  
I sistemi che utilizzano Microsoft Outlook come client di posta elettronica possono attenuare il vettore di posta elettronica HTML per Outlook 2007 e Outlook 2007 configurando la lettura dei messaggi solo in testo normale.
  
**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**  
I sistemi in cui una versione vulnerabile del modulo di gestione di database Microsoft Jet 4.0 è installata sulle edizioni supportate di Microsoft Windows 2000, Windows XP Service Pack 2 e Windows Server 2003 Service Pack 1 sono principalmente interessati da questa vulnerabilità. Questi sistemi operativi, presenti anche nella tabella Software interessato della sezione **Informazioni generali** vengono forniti con la versione vulnerabile di Jet. Pertanto, si consiglia di consultare le Domande frequenti sull'individuazione della versione vulnerabile di Jet su un sistema.
  
**Come è possibile sapere se la versione di Jet installata è vulnerabile?**  
Se il file Msjet40.dll ha un numero di versione inferiore a 4.0.9505.0, la versione di Jet del sistema è vulnerabile. La versione 4.0.9505.0 e tutte le versioni successive di Msjet40.dll non sono interessate dalla vulnerabilità di sovraccarico del buffer.
  
**Quali sono gli scopi dell'aggiornamento?**  
L'aggiornamento rimuove la vulnerabilità migliorando la convalida di strutture di dati di Jet.
  
**Al momento del rilascio di questo bollettino le informazioni sulla vulnerabilità erano disponibili pubblicamente?**  
Sì. Questa vulnerabilità è stata divulgata pubblicamente come "Vulnerabilità legata all'overflow dello stack di analisi dei file MDB del modulo di gestione di database Microsoft Jet" e identificata con il codice [CVE 2007-6026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-6026).
  
**Questo aggiornamento per la protezione risolve anche il problema identificato nell'Advisory Microsoft sulla sicurezza 950627?**  
Sì. La versione di questo aggiornamento per la protezione risolve il problema di protezione descritto nell'[Advisory Microsoft sulla sicurezza 950627](http://technet.microsoft.com/security/advisory/950627) identificato con il codice CVE 2008-1092. Dopo averlo esaminato, Microsoft ha rilevato che si tratta dello stesso problema [CVE-2007-6026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-6026) riclassificato per questo bollettino sulla sicurezza.
  
**Al momento del rilascio di questo bollettino erano già stati segnalati attacchi basati sullo sfruttamento di questa vulnerabilità?**  
Sì. Al momento del rilascio di questo bollettino, erano già stati segnalati attacchi basati sullo sfruttamento di questa vulnerabilità.
  
**L'applicazione di questo aggiornamento per la protezione consente di proteggere i clienti da codice pubblicato che tenti di sfruttare la vulnerabilità?**  
Sì. L'aggiornamento per la protezione affronta la vulnerabilità attualmente sfruttata. Questa vulnerabilità è stata identificata con il codice [CVE-2007-6026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-6026).

-   **Verifica della versione dei file**

    Poiché esistono varie edizioni di Microsoft Windows, le operazioni descritte di seguito potrebbero risultare diverse nel sistema in uso. In tal caso, consultare la documentazione del prodotto per eseguire tutti i passaggi.

    1.  Fare clic su **Start**, quindi su **Cerca**.
    2.  Nel riquadro **Risultati ricerca**, scegliere **Tutti i file e le cartelle** in **Ricerca guidata**
    3.  Nella casella **Nome del file o parte del nome**, digitare il nome del file, individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Cerca**.
    4.  Nell'elenco dei file, fare clic con il pulsante destro del mouse sul file individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Proprietà**.
        **Nota** A seconda dell'edizione del sistema operativo o dei programmi installati nel sistema, è possibile che alcuni dei file elencati nella tabella di informazioni sui file non siano installati.
    5.  Nella scheda **Versione** individuare il numero di versione del file installato nel sistema e confrontarlo con quello specificato nell'appropriata tabella di informazioni sui file.
        **Nota** È possibile che durante l'installazione alcuni attributi diversi dalla versione del file vengano modificati. Il confronto di altri attributi dei file con le informazioni disponibili nella tabella non è un metodo supportato per la verifica dell'applicazione dell'aggiornamento. In alcuni casi, inoltre, è possibile che i file vengano rinominati durante l'installazione. Se le informazioni sul file o sulla versione non sono disponibili, utilizzare uno degli altri metodi disponibili per verificare l'installazione dell'aggiornamento.

-   **Controllo delle chiavi del Registro di sistema**

    Per verificare i file installati dall'aggiornamento per la protezione, è anche possibile esaminare le chiavi del Registro di sistema elencate nella **Tabella di riferimento** in questa sezione.

    Poiché esistono varie edizioni di Microsoft Windows, le operazioni descritte di seguito potrebbero risultare diverse nel sistema in uso. In tal caso, consultare la documentazione del prodotto per eseguire tutti i passaggi.

    1.  Fare clic su **Start**, quindi su **Cerca**.
    2.  Nel riquadro **Risultati ricerca**, scegliere **Tutti i file e le cartelle** in **Ricerca guidata**
    3.  Nella casella **Nome del file o parte del nome**, digitare il nome del file, individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Cerca**.
    4.  Nell'elenco dei file, fare clic con il pulsante destro del mouse sul file individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Proprietà**.

    5.  Nella scheda **Versione** individuare il numero di versione del file installato nel sistema e confrontarlo con quello specificato nell'appropriata tabella di informazioni sui file.
        **Nota** È possibile che durante l'installazione alcuni attributi diversi dalla versione del file vengano modificati. Il confronto di altri attributi dei file con le informazioni disponibili nella tabella non è un metodo supportato per la verifica dell'applicazione dell'aggiornamento. **Verifica dell'applicazione dell'aggiornamento**

-   **Microsoft Baseline Security Analyzer**

    Per verificare se un aggiornamento per la protezione è stato applicato al sistema interessato, è possibile utilizzare lo strumento Microsoft Baseline Security Analyzer (MBSA). Per ulteriori informazioni, vedere la sezione Strumenti e informazioni sul rilevamento e sulla distribuzione nel presente bollettino.

-   **Verifica della versione dei file**

    Poiché esistono varie edizioni di Microsoft Windows, le operazioni descritte di seguito potrebbero risultare diverse nel sistema in uso. In tal caso, consultare la documentazione del prodotto per eseguire tutti i passaggi.

    1.  Fare clic su **Start**, quindi su **Cerca**.
    2.  Nel riquadro **Risultati ricerca**, scegliere **Tutti i file e le cartelle** in **Ricerca guidata**
    3.  Nella casella **Nome del file o parte del nome**, digitare il nome del file, individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Cerca**.
    4.  Nell'elenco dei file, fare clic con il pulsante destro del mouse sul file individuato nell'appropriata tabella di informazioni sui file, quindi fare clic su **Proprietà**.
        **Nota** A seconda dell'edizione del sistema operativo o dei programmi installati nel sistema, è possibile che alcuni dei file elencati nella tabella di informazioni sui file non siano installati.
    5.  Nella scheda **Versione** individuare il numero di versione del file installato nel sistema e confrontarlo con quello specificato nell'appropriata tabella di informazioni sui file.
        **Nota** È possibile che durante l'installazione alcuni attributi diversi dalla versione del file vengano modificati. Il confronto di altri attributi dei file con le informazioni disponibili nella tabella non è un metodo supportato per la verifica dell'applicazione dell'aggiornamento. In alcuni casi, inoltre, è possibile che i file vengano rinominati durante l'installazione. Se le informazioni sul file o sulla versione non sono disponibili, utilizzare uno degli altri metodi disponibili per verificare l'installazione dell'aggiornamento.

-   **Controllo delle chiavi del Registro di sistema**

    Per verificare i file installati dall'aggiornamento per la protezione, è anche possibile esaminare le chiavi del Registro di sistema elencate nella **Tabella di riferimento** in questa sezione.

    Queste chiavi del Registro di sistema potrebbero non contenere un elenco completo dei file installati. Inoltre, se un amministratore o un OEM integra o esegue lo slipstreaming dell'aggiornamento per la protezione nei file di origine dell'installazione di Windows, è possibile che queste chiavi del Registro di sistema non vengano create correttamente.
