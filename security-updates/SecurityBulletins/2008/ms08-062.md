Security Bulletin

Bollettino Microsoft sulla sicurezza MS08-062 - Importante
==========================================================

Una vulnerabilità del servizio di stampa Internet di Windows può consentire l'esecuzione di codice in modalità remota (953155)
------------------------------------------------------------------------------------------------------------------------------

Data di pubblicazione: martedì 14 ottobre 2008 | Aggiornamento: mercoledì 29 ottobre 2008

**Versione:** 2.2

### Informazioni generali

#### Riepilogo

L'aggiornamento risolve una vulnerabilità segnalata privatamente del servizio di stampa Internet di Windows che può consentire l'esecuzione di codice in modalità remota. Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe assumere il controllo completo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account.

Questo aggiornamento è considerato di livello importante per tutte le edizioni supportate di Microsoft Windows 2000, Windows XP, Windows Server 2003 e Windows Server 2008. Per ulteriori informazioni, vedere la sottosezione **Software interessato e Software non interessato** all'interno della presente sezione.

Questo aggiornamento per la protezione risolve la vulnerabilità modificando il modo in cui viene gestita l'allocazione della memoria nel servizio IPP (Internet Printing Protocol). Per ulteriori informazioni sulla vulnerabilità, vedere la sottosezione Domande frequenti per la specifica voce della vulnerabilità nella sezione **Informazioni sulle vulnerabilità**.

**Raccomandazione.** Microsoft consiglia di installare l'aggiornamento quanto prima.

**Problemi noti.** Nessuno

Domande frequenti sull'aggiornamento per la protezione
------------------------------------------------------

<span></span>
**Perché Microsoft ha rivisto questo bollettino in data 29 ottobre 2008?**  
Microsoft ha rivisto questo bollettino per avvisare i clienti che i pacchetti di aggiornamento per Windows Server 2008 per i sistemi basati su Itanium e per tutte le edizioni supportate di Windows Vista sono ora disponibili su Windows Update, Microsoft Update, Windows Server Update Services (WSUS), Systems Management Server e System Center Configuration Manager. Al momento del rilascio di questo bollettino gli aggiornamenti per queste piattaforme erano disponibili solo nell'Area download Microsoft.

**Perché Microsoft ha rivisto questo bollettino in data 16 ottobre 2008?**
Microsoft ha rivisto questo bollettino per chiarire che il servizio di stampa Internet di Windows viene eseguito nel contesto del servizio Spooler il quale, a sua volta, viene eseguito con i privilegi del sistema. Un utente malintenzionato in grado di sfruttare la vulnerabilità descritta in questo bollettino sulla sicurezza potrebbe eseguire codice dannoso nel sistema di un utente con privilegi di sistema e assumere il controllo completo del sistema interessato.

**Perché Microsoft ha rivisto questo bollettino in data 15 ottobre 2008?**
Microsoft ha rivisto questo bollettino al fine di rimuovere il livello di gravità per Windows Server 2008 per sistemi basati su Itanium. Inoltre, Microsoft ha aggiornato questo bollettino al fine di chiarire che l'aggiornamento per Windows Server 2008 per sistemi basati su Itanium è disponibile tramite l'Area download Microsoft e sarà rilasciato attraverso gli altri canali di distribuzione al più presto possibile.

**Perché Windows Server 2008 per i sistemi basati su Itanium e Windows** **Vista** **sono menzionati nella tabella Software interessato se non presentano alcun livello di gravità?**
Windows Server 2008 per i sistemi basati su Itanium e tutte le edizioni supportate di Windows Vista contengono il componente spooler di stampa condiviso, che viene aggiornato dal presente aggiornamento per la protezione. Stampa Internet tuttavia, vale a dire il vettore vulnerabile trattato nel presente bollettino sulla sicurezza, non è supportata da Windows Server 2008 per i sistemi basati su Itanium e da Windows Vista. Agli utenti che utilizzano Windows Server 2008 per sistemi basati su Itanium e Windows Vista si consiglia di applicare il presente aggiornamento per la protezione come misura preventiva per difendersi da eventuali nuovi vettori identificabili in futuro.

Vulnerabilità legata all'overflow di valori integer nel servizio IPP - CVE-2008-1446  
------------------------------------------------------------------------------------
  
<span></span>
Esiste una vulnerabilità legata all'esecuzione di codice in modalità remota nei sistemi Windows che eseguono IIS con il servizio di stampa Internet attivato. Questo problema può consentire a un utente malintenzionato remoto e autenticato di eseguire del codice arbitrario su un sistema interessato.
  
Per visualizzare questa vulnerabilità come voce standard nell'elenco CVE, vedere il codice [CVE-2008-1446](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1446).
  
#### Fattori attenuanti della vulnerabilità legata all'overflow di valori integer nel servizio IPP - CVE-2008-1446
  
Per fattore attenuante si intende un'impostazione, una configurazione comune o una procedura consigliata generica esistente in uno stato predefinito in grado di ridurre la gravità nello sfruttamento di una vulnerabilità. I seguenti fattori attenuanti possono essere utili per l'utente:
  
-   Il protocollo IPP (Internet Printing Protocol) è attivato per impostazione predefinita in IIS per Microsoft Windows 2000 e Windows XP. Tuttavia, le autorizzazioni predefinite dei file di tutte le piattaforme sono impostate in modo che sia possibile accedere a IPP soltanto agli utenti del gruppo "Autenticati". A causa di questa restrizione, gli utenti non autenticati per impostazione predefinita non disporranno delle autorizzazioni per le cartelle necessarie a sfruttare questa vulnerabilità in modalità remota.  
-   Le installazioni di IIS su Windows Server 2003 non sono vulnerabili nella loro configurazione predefinita. Sono vulnerabili soltanto le installazioni su Windows Server 2003 che hanno attivato il filtro ISAPI del protocollo IPP all'interno di IIS. Entrambi i componenti Stampa Internet e Pagine ASP sono necessari per attivare la gestione delle stampanti basata sul Web e la stampa Internet. Se il componente Pagine ASP è impostato su "Consentito" e il componente Stampa Internet è impostato su "Non consentito", è attivata soltanto la gestione delle stampanti basata sul Web. Gli utenti non possono quindi connettersi ad alcuna stampante condivisa mediante Stampa Internet.  
-   Le installazioni di IIS su Windows Server 2008 non sono vulnerabili per impostazione predefinita. Un computer che esegue Windows Server 2008 può agire come un client che utilizza Stampa Internet. Tuttavia, prima di poter utilizzare una qualsiasi stampante connessa al server di stampa, gli utenti del computer che invia le richieste di stampa devono essere autenticati da tale server. Per attivare un computer che esegue Windows Server 2008 affinché funga da server supportando la stampa Internet, è necessario installare i ruoli Server Web (IIS), Servizi di stampa e il servizio ruolo Stampa Internet nel ruolo Servizi di stampa.  
-   Per poter sfruttare questa vulnerabilità il server IIS deve comunicare in uscita con un computer dannoso sulla porta 445. Gli amministratori che bloccano le connessioni SMB in uscita dal loro server IIS verso Internet possono impedire che questa vulnerabilità venga sfruttata da sistemi non attendibili.
  
#### Soluzioni alternative per la vulnerabilità legata all'overflow di valori integer nel servizio IPP - CVE-2008-1446
  
Per soluzione alternativa si intende un'impostazione o una modifica nella configurazione che non elimina la vulnerabilità sottostante, ma consente di bloccare gli attacchi noti prima di applicare l'aggiornamento. Microsoft ha verificato le seguenti soluzioni alternative e segnala nel corso della discussione se tali soluzioni riducono la funzionalità:
  
-   **Disattivazione del servizio IPP**
  
    Per disattivare il servizio IPP in IIS 5.0 e IIS 5.1
  
    1.  Aprire **Gestione servizi Internet**.  
    2.  Fare clic con il pulsante destro del mouse su **Sito Web predefinito** e scegliere **Proprietà**.  
    3.  Aprire la scheda **Home Directory** e selezionare **Configurazione**.  
    4.  Scorrere tutto l'elenco per trovare l'estensione della **stampante** e il percorso corrispondente a **msw3prt.dll**.  
    5.  Fare clic su **Rimuovi**.  
    6.  Fare clic su **OK**.  
    7.  Riavviare il servizio IIS.
  
    **Nota** Stampa Internet può essere configurata tramite Criteri di gruppo oppure Gestione servizi Internet. In caso di conflitto tra le impostazioni dei Criteri di gruppo e quelle di Gestione servizi Internet, le prime avranno la precedenza. Se si rimuove Stampa Internet tramite Gestione servizi Internet, verificare che il servizio non venga riattivato mediante i criteri di gruppo locali o di dominio (i criteri di gruppo predefiniti non attivano né disattivano Stampa Internet). Nello snap-in MMC Criteri di gruppo, verificare Configurazione computer | Modelli amministrativi | Stampa | Stampa basata su Web.
  
    Per disattivare il servizio IPP in IIS 6.0
  
    1.  Avviare **Gestione IIS** o lo **snap-in IIS**.  
    2.  Espandere **\* nome\_server**, dove nome\_server è il nome del server.  
    3.  Fare clic su **Estensioni del servizio Web.**  
    4.  Nel riquadro a destra fare clic su **Stampa Internet**, quindi scegliere **Non consentito**.  
    5.  Uscire da Gestione IIS.
  
    Per disattivare il servizio IPP in IIS 7.0 su Windows Server 2008
  
    1.  Se **ServerManager** non è in esecuzione, fare clic su **Start**, scegliere **Strumenti di amministrazione**, quindi fare clic su **Server Manager**. Se si apre la finestra di dialogo **Controllo account utente**, verificare che l'azione visualizzata corrisponda a quella desiderata, quindi fare clic su **Continua**.  
    2.  Espandere il nodo Ruoli, fare clic con il pulsante destro del mouse su **Servizi di stampa** e selezionare **Rimuovi servizi di ruolo**.  
    3.  Verrà visualizzata la procedura guidata Rimuovi servizi di ruolo. Se è selezionata l'opzione **Stampa Internet**, deselezionare la casella di controllo e fare clic su **Avanti**, quindi selezionare **Rimuovi** per completare la procedura guidata.
  
    **Effetto della soluzione alternativa.** Gli utenti non potranno connettersi ad alcuna stampante condivisa mediante Stampa Internet.
  
    **Per annullare il risultato della soluzione alternativa.**
  
    Per disattivare il servizio IPP in IIS 5.0 e IIS 5.1
  
    1.  Aprire **Gestione servizi Internet**.  
    2.  Fare clic con il pulsante destro del mouse su **Sito Web predefinito** e scegliere **Proprietà**.  
    3.  Aprire la scheda **Home Directory** e selezionare **Configurazione**.  
    4.  Scorrere tutto l'elenco per trovare l'estensione della **stampante** e il percorso corrispondente a **msw3prt.dll**.  
    5.  Fare clic su **Aggiungi**.  
    6.  Nel campo denominato **Eseguibile**, immettere quanto segue:  
        %systemroot%\\system32\\msw3prt.dll  
    7.  Nel campo denominato **Estensione**, immettere quanto segue:  
        .printer  
    8.  Nel campo denominato **Limita a**, immettere quanto segue:  
        GET,POST  
    9.  Dopo aver effettuato queste operazioni, riavviare il servizio IIS.
  
    Per attivare il servizio IPP in IIS 6.0
  
    1.  Avviare **Gestione IIS** o lo **snap-in IIS**.  
    2.  Espandere **\* nome\_server**, dove nome\_server è il nome del server.  
    3.  Fare clic su **Estensioni del servizio Web.**  
    4.  Nel riquadro a destra fare clic su **Stampa Internet**, quindi scegliere **Consenti**.  
    5.  Uscire da Gestione IIS.
  
    Per attivare il servizio IPP in IIS 7.0
  
    1.  Se **ServerManager** non è in esecuzione, fare clic su **Start**, scegliere **Strumenti di amministrazione**, quindi fare clic su **Server Manager**. Se si apre la finestra di dialogo **Controllo account utente**, verificare che l'azione visualizzata corrisponda a quella desiderata, quindi fare clic su **Continua**.  
    2.  Espandere il nodo Ruoli, fare clic con il pulsante destro del mouse su **Servizi di stampa** e selezionare **Aggiungi servizi di ruolo**.  
    3.  Verrà visualizzata la procedura guidata Aggiungi servizi di ruolo. Selezionare l'opzione **Stampa Internet**, fare clic su **Avanti**, quindi su **Installa** per completare la procedura guidata.
  
-   **Eseguire lo strumento IIS Lockdown 2.1.**
  
    In IIS 5.0 eseguito in Microsoft Windows 2000 o in IIS 5.1 eseguito in Windows XP, IIS Lockdown Wizard versione 2.1 può essere utilizzata per disattivare le funzionalità non necessarie, inclusa Stampa Internet. Per eseguire questa procedura guidata, effettuare le operazioni seguenti:
  
    1.  Scaricare lo strumento ISS Lockdown 2.1 da [qui](http://www.microsoft.com/downloads/details.aspx?familyid=dde9efc0-bb30-47eb-9a61-fd755d23cdec&displaylang=en).  
    2.  Effettuare una delle operazioni seguenti:  
        Per avviare immediatamente l'installazione, fare clic su **Apri** o **Esegui l'applicazione dalla posizione corrente**.  
        Per copiare il download sul computer ed eseguire l'installazione successivamente, fare clic su **Salva** o **Salva l'applicazione su disco**.
  
    **Impatto della soluzione alternativa.**
  
    Gli utenti non potranno connettersi ad alcuna stampante condivisa mediante Stampa Internet. Inoltre, gli utenti dello strumento IIS Lockdown non potranno connettersi ad alcuna stampante condivisa mediante Stampa Internet. Per ottenere un elenco completo delle modifiche apportate dallo strumento IIS Lockdown, vedere il seguente [articolo di MSDN](http://msdn.microsoft.com/en-us/library/aa302365,aspx).
  
    **Per annullare il risultato della soluzione alternativa.**
  
    Per annullare le modifiche apportate da ISS Lockdown, eseguire **IISlockd.exe** una seconda volta e scegliere di annullare le modifiche. In tal modo sarà possibile effettuare immediatamente un eventuale annullamento.
  
    **Nota** Il filtro ISAPI URLScan installato con lo strumento IIS Lockdown non viene rimosso durante il processo di annullamento. È possibile rimuovere URLScan manualmente utilizzando la scheda Filtri ISAPI a livello di server in Gestione servizi Internet.
  
#### Domande frequenti relative alla vulnerabilità legata all'overflow di valori integer nel servizio IPP - CVE-2008-1446
  
**Qual è la portata o l'impatto di questa vulnerabilità?**    
Si tratta di una vulnerabilità legata all'esecuzione di codice in modalità remota. Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe assumere il controllo completo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account.
  
**Quali sono le cause di questa vulnerabilità?**    
La funzione IPP è implementata come un'estensione ISAPI per IIS. Esiste una vulnerabilità remota autenticata nell'implementazione di Microsoft Internet Printing Protocol (IPP) su server Windows che eseguono IIS. Si verifica un overflow di valori interger nell'implementazione di IPP che può consentire a un utente malintenzionato autenticato di eseguire codice in modalità remota sui server IIS interessati.
  
**Che cosa significa ISAPI?**  
ISAPI (Internet Services Application Programming Interface) è una tecnologia che consente agli sviluppatori Web di ampliare le funzionalità dei loro server Web scrivendo codice personalizzato che fornisce nuovi servizi a un server Web. Il codice personalizzato può essere implementato in un filtro ISAPI, se la nuova funzionalità fornisce un servizio di basso livello, o in un'estensione ISAPI, se la nuova funzionalità fornisce un servizio di alto livello. In questo caso, il codice interessato è un'estensione ISAPI.
  
**Quale tipo di estensione ISAPI è interessata?**  
L'estensione ISAPI interessata è quella che implementa il protocollo IPP (Internet Printing Protocol), uno standard di settore definito dalle RFC 2910 e 2911. Il protocollo IPP consente di richiedere servizi di stampa e conoscere lo stato dei processi di stampa attraverso Internet via HTTP. Ad esempio, utilizzando il protocollo IPP un utente mobile può avviare un processo di stampa via Internet e stampare su una stampante della propria rete aziendale. L'utente può inoltre verificare se la richiesta di stampa è stata completata senza errori.
  
Microsoft Windows 2000 ha introdotto il supporto nativo per Stampa Internet. L'implementazione in Microsoft Windows 2000 consente agli utenti di stampare direttamente su un URL e di visualizzare informazioni sui processi di stampa mediante il browser. Il supporto per Stampa Internet è attivato per impostazione predefinita nelle versioni server di Microsoft Windows 2000. Può essere installato come componente facoltativo anche in Microsoft Windows 2000 Professional, Windows XP, Windows Server 2003 e Windows Server 2008.
  
**A quali attacchi viene esposto il sistema a causa di questa vulnerabilità?**    
Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe eseguire codice non autorizzato nel sistema di un utente con privilegi di sistema. Sfruttando questa vulnerabilità, un utente malintenzionato potrebbe assumere il controllo completo del sistema interessato. Potrebbe quindi installare programmi e visualizzare, modificare o eliminare dati oppure creare nuovi account.
  
**In quale modo un utente malintenzionato può sfruttare questa vulnerabilità?**    
Un utente malintenzionato potrebbe inviare una richiesta HTTP POST appositamente concepita a un server Web vulnerabile, che si connetterebbe così a un computer controllato dall'utente malintenzionato che agisce come una stampante utilizzando il protocollo IPP. L'utente malintenzionato utilizzerebbe quindi risposte appositamente predisposte per eseguire il codice in modalità remota sul server di destinazione.
  
**Quali sono i sistemi principalmente interessati da questa vulnerabilità?**    
I sistemi più esposti sono le workstation e i server che eseguono IIS con il servizio Internet Printing Protocol (IPP) attivato.
  
**Quali sono gli scopi dell'aggiornamento?**    
Questo aggiornamento per la protezione risolve la vulnerabilità convalidando i dati ricevuti dai computer remoti nel servizio Internet Printing Protocol (IPP).
  
**Al momento del rilascio di questo bollettino le informazioni sulla vulnerabilità erano disponibili pubblicamente?**    
No. Microsoft ha ricevuto informazioni sulla vulnerabilità da fonti private. Secondo le informazioni in possesso di Microsoft, al momento della pubblicazione del presente bollettino l'esistenza di questa vulnerabilità non era stata ancora divulgata pubblicamente.
  
**Al momento del rilascio di questo bollettino erano già stati segnalati attacchi basati sullo sfruttamento di questa vulnerabilità?**    
Sì, Microsoft è a conoscenza di attacchi mirati che tentano di sfruttare questa vulnerabilità. Tuttavia, al momento della pubblicazione del presente bollettino, Microsoft non è a conoscenza dell'esistenza di un codice di prova pubblicato.

    3.  Nella scheda **Generale**, confrontare le dimensioni del file con le tabelle di informazioni sui file fornite nell'articolo della Knowledge Base indicato nel bollettino.
